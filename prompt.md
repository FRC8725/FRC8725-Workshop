# 功能需求：Data Matrix 標籤輸出與相機掃描取用

請先理解並沿用現有專案的資料模型、權限機制、操作流程與視覺風格，再完成以下功能。重點是依照既有系統的邏輯整合需求，不要為此建立互相重複的物品資料或另一套取用流程。

本需求中提到的「條碼」一律是 **Data Matrix**，不是 QR Code 或其他一維／二維條碼。Data Matrix 的產生與解碼都必須採用可在前端執行的 JavaScript 函式庫，不可依賴瀏覽器原生條碼偵測 API。選用的方案必須能兼容主流桌面與行動裝置瀏覽器，並在不支援原生條碼 API 的設備上正常運作。

## 一、在除錯模式輸出櫃內工具標籤 PDF

僅當網址查詢參數包含 `debug=1` 時，在櫃子內容頁提供「輸出 Data Matrix」操作；一般模式不可顯示或開放此功能。

使用者觸發輸出後，系統應根據目前櫃子的即時資料產生一份可直接列印的 A4 PDF，規則如下：

- 只包含目前櫃子內類型為「工具」的物品，不包含材料或其他類型。
- 每個 Data Matrix 的編碼內容只能是該工具的 `id`。
- 依工具的「總數量」決定標籤份數。例如總數量為 5，便產生 5 個內容完全相同的 Data Matrix；總數量為 0 時不產生標籤。
- 每個 Data Matrix 的實際列印尺寸為 `1 cm × 1 cm`，不可因頁面縮放、排版或不同裝置而改變。
- 條碼四周不顯示物品名稱、ID、數量、位置或其他文字。若解碼規格需要靜區，應保留可可靠掃描的最小必要靜區，但不得加入無關裝飾或資訊。
- 標籤在 A4 可列印範圍內緊湊排列，列與欄的間距一致，兼顧紙張利用率與剪裁便利性。
- 條碼的排列順序依照名稱，由左至右、由上至下
- 以工具所在的位置區塊分組，例如上層、中層、下層或系統現有的抽屜／層架區域。不同位置群組之間必須有清楚且適合剪裁的分隔線；同一位置的標籤應保持相鄰，不可與其他位置交錯。
- 分組順序應遵循櫃子目前的位置顯示順序，而不是任意依名稱或 ID 排序。若某個位置沒有工具，略過該區塊。
- 分隔線本身用來表達群組邊界，不應侵入 Data Matrix 或其必要靜區。跨頁時不得讓條碼被裁切；必要時將完整標籤或完整區段移至下一頁。
- PDF 中的 Data Matrix 必須以適合列印的高解析度或無失真形式呈現，在 100%／實際大小列印時仍可穩定掃描。
- 若櫃內沒有可輸出的工具，應明確告知使用者，且不要產生空白 PDF。
- 產生期間應避免重複觸發；完成後提供明確結果，失敗時顯示可理解的錯誤，且不影響櫃子原有操作。

## 二、在作業區以相機掃描工具並加入待取用清單

此功能在一般模式即可使用。在作業區的「取用物品」流程中，保留既有的手動輸入／搜尋方式，另外提供「開啟相機」操作，讓使用者掃描工具上的 Data Matrix。

掃描流程應遵循以下邏輯：

1. 開啟相機前先處理相機權限；啟用後提供清楚的掃描狀態與取景畫面。
2. 持續從相機影像中辨識 Data Matrix，但同一個條碼在短時間內連續出現在畫面時，只能視為一次有效掃描，避免單次對準造成重複加入。
3. 解碼結果視為物品 `id`，以此查找系統中的物品資料，不把掃描內容直接當作名稱或可信資料使用。
4. 只有存在、類型為工具、目前可取用且符合既有取用規則的物品，才能加入待取用清單。
5. 每次有效掃描代表新增該工具 1 件至待取用清單。若同一工具再次被使用者刻意掃描，可依既有清單呈現方式增加數量或增加一筆，但待取用總量不得超過該工具目前的可用數量。
6. 成功後立即提供視覺回饋，並自動更新待取用清單；掃描本身不可立即扣除庫存，也不可直接完成取用。
7. 使用者可以繼續掃描其他工具，也能沿用手動方式增刪或調整待取用清單。
8. 只有在使用者完成挑選並按下既有的「全部放入盒子」後，才依現有流程一次確認並提交整份清單。

以下狀況必須有清楚且不會中斷整個取用流程的提示：

- 相機權限遭拒、裝置沒有相機，或相機無法啟動。
- 條碼無法辨識、內容為空、格式不合法，或找不到對應物品。
- 掃描到材料、不可取用的工具、可用數量不足，或加入後會超過可用數量。
- 提交前庫存已被其他操作改變；此時應依最新資料重新驗證，避免數量變成負值或產生超額取用。

關閉掃描介面、切換頁面或完成操作時，應停止相機並釋放資源。掃描介面需適合手機直向與橫向使用，也不可破壞桌面版既有操作。

## 共通品質要求

- Data Matrix 的生成器與解碼器必須使用 JavaScript 函式庫，並以跨裝置兼容性、Data Matrix 支援完整度、維護狀態及掃描可靠性作為選擇依據。
- 不可使用僅支援 QR Code 的方案，也不可把瀏覽器原生 Barcode Detection API 當作必要依賴。
- 沿用現有物品 ID、數量、位置、狀態與權限規則，避免新增不一致的平行資料來源。
- 所有新文字、按鈕、載入狀態及錯誤訊息應符合現有介面的語言與視覺規範。
- 不得影響既有的手動取用、櫃子瀏覽、物品編輯及非除錯模式行為。

## 驗收條件

- `?debug=1` 下可從指定櫃子輸出該櫃所有工具的 Data Matrix PDF；沒有此參數時看不到該操作。
- 每項工具產生的條碼數等於其總數量，且每個條碼只編碼該物品 ID。
- PDF 為 A4、條碼實際尺寸為 1 cm × 1 cm、間距一致、可剪裁、依位置清楚分段，且列印後可掃描。
- 一般模式下可用相機辨識 Data Matrix，並正確找到對應工具及加入 1 件至待取用清單。
- 相機掃描與手動選取可以混合使用，且只有按下「全部放入盒子」才真正提交取用。
- 連續畫面不會誤把同一個條碼加入多次；使用者再次有意掃描時則能新增，但不會超過可用庫存。
- 權限拒絕、未知 ID、錯誤條碼、非工具、庫存不足和相機不可用等情況都有合理回饋，且使用者仍可改用手動流程。
- 在主流桌面與行動裝置瀏覽器中，不依賴原生條碼 API 也能完成 Data Matrix 的產生與解碼。
